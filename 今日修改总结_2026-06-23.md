# 今日修改总结 2026-06-23

本次提交包含两个主要目录：

- `带采集CPU121XC3B10C智能化/`：TIA Portal V21 项目文件夹。
- `分析/`：DB、FB、FC、OB 文本源、Modbus 采集新增文件、变量表和修改说明。

## 主要目标

在三部十层电梯工程基础上，增强采集、智能调度、初始化、检修/超载安全处理相关逻辑，使工程更适合 EET Pro 场景采集、Python 智能调度联调和比赛报告分析。

## 已修改或重点整理的模块

### `FB7_单梯控制`

- 新增每部电梯的 `梯号` 输入，用于按梯号读取独立参数。
- 称重调用改为使用 `DB_系统配置.梯满载阈值[#梯号]` 和 `DB_系统配置.梯超重阈值[#梯号]`。
- 新增安全停靠逻辑：
  - `检修信号 OR 超载信号` 触发安全停靠。
  - 运行中只请求停层，不直接开门。
  - 等待电梯到达平层且上下行接触器断开后，再置位到站开门服务。
- 调整检修最终钳制，避免检修信号刚出现时破坏安全停靠过程。

### `FB2_检修控制`

- 检修信号进入时，如果电梯仍在运行，不再直接切断全部运动输出，而是交给 `FB7/FB12` 完成安全停靠。
- 检修手动开门增加平层静止条件，防止非平层开门。
- 超载状态下禁止检修手动关门。

### `FB6_开关门控制`

- 检修模式下，如果安全停靠已经置位到站服务/手动开门服务，不再提前 `RETURN`，允许开门逻辑执行。
- 保留超载禁止关门逻辑。

### `FB12_运动控制`

- 检修信号有效但电梯仍在运行时，不再提前退出运动控制。
- 允许继续使用原有平层停车、制动锁存、低速/高速控制逻辑完成安全停靠。

### `OB1`

- 单梯控制改为显式调用三部电梯，避免配置值异常导致只调用一部电梯。
- 调整检修内部安全钳制：检修信号有效但方向接触器仍在运行时，不直接清运行输出，等安全停靠完成后再清理显示/制动输出。

### `FC_输出映射`

- 调整检修最终输出保险：
  - 检修信号有效但方向接触器仍在安全停靠运行中时，不在输出层直接断开。
  - 等方向接触器断开后再清理最终运行输出。
- 保留检修状态故障灯显示逻辑。

### `FB15_群控调度`

- 消号冷却从全局冷却改为按楼层、按方向独立冷却。
- 避免一个楼层开门消号后，其他楼层外呼在冷却期内丢失。
- 超时重分配从扫描次数计数改为 `TON_TIME` 秒脉冲。
- 保留 Python 智能调度策略分支和原就近调度回退能力。

### `FB10_本层停靠判断`

- 新增离开确认机制。
- 电梯刚从某层起步但尚未真正离开平层区时，不允许被同层请求重新抓回。
- 避免“刚起步又被本层外呼/内呼重新捕获”的停靠死锁。

### `FB11_称重数据分析`

- 保留模拟量称重计算。
- 支持老版 EET 离散超重信号：当模拟量为 0 时，不覆盖 `FC_输入映射` 写入的超载信号。
- 满载阈值和超重阈值由 `FB7` 按梯号传入。

### `DB_系统配置`

- 增加每部电梯独立阈值：
  - `梯超重阈值 : Array[1..3] of Real`
  - `梯满载阈值 : Array[1..3] of Real`
- 保留原有全局阈值字段，兼容旧逻辑和说明。

### Modbus 采集相关文件

- 保留并整理 `Modbus采集新增/` 下的采集映射、生成脚本和寄存器说明。
- 当前采集区用于 Python 读取 PLC 状态和写回智能调度建议。
- 智能调度策略通过 `DB_群控状态.智能调度允许` 和 `DB_系统配置.群控调度策略` 控制启用/回退。
