# 样题功能实现对照审查

日期：2026-06-24

对照文件：

- `1 - 2026年CIMC离散行业自动化（逻辑算法）初赛 - 样题6390795519224792462993263.pdf`
- 已抽取文本：`sample_pdf_extracted.txt`
- 程序源：`TXT 文件等`、`DB 模块`

## 总结

当前程序不是完整实现样题全部功能。

已经具备的核心基础：

- 三部电梯基本输入/输出映射。
- 初始化、准备就绪、检修退出重新初始化。
- 内呼、外呼登记、群控分配。
- 开关门控制、光幕打断关门、门动作超时。
- 起停控制、上下端站限位、三级减速制动。
- 楼层显示、上下行方向指示。
- 满载过滤外呼、超载禁止关门/启动。
- 待载照明/风扇延时。
- Python 智能帧采集/写回、策略 2 智能调度接入。

明显缺失或不完整：

- 消防应急模式。
- 独立运行/司机模式。
- 锁梯与远程禁用。
- 动态分区调度。
- 群控效能评估，如运行效率、空载率、系统能耗指标。
- PLC 与 EET 模型通讯故障保护。
- 运行超时保护。
- 开关门故障隔离逻辑不完整。
- 超载故障灯闪烁不完整。
- WINCC 画面、权限、报警、检修时间记录等 HMI 要求未体现。

## 功能对照

| 样题功能 | 当前状态 | 程序证据/说明 |
|---|---|---|
| 电梯初始化 | 基本实现 | `FB8_双向初始化.txt` 处理自动运行、手动触发、检修退出后重新初始化，并置位 `准备就绪`。 |
| 集选控制 | 基本实现 | `FB9_呼梯管理` 处理内呼；`FB15_群控调度` 登记外呼并分配到单梯；`FC_需求汇总` 汇总需求。 |
| 开关门控制 | 部分实现 | `FB6_开关门控制` 有光幕、开门按钮、自动关门、开关门超时；但超时后没有完整故障锁存/群控隔离。 |
| 启停控制 | 基本实现 | `FB1_起动过程` 起动条件包含门锁、关门到位、非超载；`FB12_运动控制` 有三级制动和端站停车。 |
| 运行监控与显示 | 基本实现 | `FB4 楼层显示`、`FB3_方向指示控制`、`FC_输出映射`。 |
| 错误指令消除 | 不完整 | 目前只看到通用按钮和部分消号冷却，没有完整“双击取消、反向登录禁止、到最远端清后方选层”的明确逻辑。 |
| 待载休眠 | 基本实现 | `FB13_照明通风` 结合需求、载重、门状态、检修/故障控制照明风扇。 |
| 满载直驶 | 部分实现 | `FB10_本层停靠判断` 对外呼停靠有 `NOT 满载指示`；`FB15` 过滤满载梯。但“只驶向轿内最近目标楼层”的专门策略不明显。 |
| 消防应急模式 | 缺失 | 程序中未发现 `消防` 相关变量或逻辑。 |
| 独立运行/司机模式 | 缺失 | 程序中未发现 `独立`、`司机` 模式变量或逻辑。 |
| 超载保护 | 部分实现 | `FB11` 置位 `超载信号`，`FB1` 禁止起动，`FB6` 禁止关门；但样题要求故障灯闪烁，当前未见闪烁逻辑。 |
| 终端越程保护 | 基本实现 | `FB12` 检测上/下端站限位并强制停车。 |
| 开关门保护 | 部分实现 | 有关门/开门 8 秒超时，但未完整实现“门锁未闭合故障常亮、光幕始终不消失则故障并分离出群控”。 |
| 运行保护 | 部分实现 | `FB6` 在离开平层或运行时强制清门继电器；但异常状态报警/锁存不完整。 |
| 通讯故障保护 | 缺失 | 只有 Python 智能帧心跳失效降级，不是 PLC 与 EET 模型通讯故障保护。没有就近平层开门、WINCC 报警、人工确认恢复逻辑。 |
| 运行超时保护 | 缺失 | 未发现楼层间最大运行时间、运行超时故障锁存。 |
| 群控调度 | 部分实现 | 有传统就近调度和 Python 策略 2；但不包含锁梯、远程禁用、动态分区、全系统停运报警。 |
| 智能交通模式识别 | PLC 接口已接入，完整性依赖 Python | PLC 已接收 `Python主场景` 和智能帧；Python 侧已有模型框架。WINCC 实时展示未实现。 |
| 能效分析与优化 | 部分实现 | Python/Modbus 可采集运行距离、动作位图等数据；PLC 侧无实时能耗指标、空载率、运行效率统计和展示。 |
| 检修模式 | 基本实现 | `FB2_检修控制` 进入检修强制停车、开门、置故障指示；退出后 `FB8` 可重新初始化。 |
| 多部电梯同时故障降级 | 缺失 | 未发现“仅剩少数电梯可用时调整策略/全部故障停运报警”。 |
| WINCC 监控画面 | 未审查/未体现 | 当前文件夹主要是 PLC TXT/DB 源，没有 HMI 画面源，不能证明满足画面、权限、报警记录要求。 |

## 最需要补的优先级

1. 消防应急模式。
2. 运行超时保护。
3. 通讯故障保护。
4. 开关门故障锁存与群控隔离。
5. 独立运行/司机模式。
6. 锁梯/远程禁用。
7. 动态分区。
8. 能耗、运行效率、空载率统计。
9. WINCC 展示、报警、权限、检修时间记录。

## 对当前智能调度部分的判断

智能调度链路已经具备最小闭环：

- PLC 采集 `0..39`。
- Python 写回 `40..79`。
- `FB_智能帧验证` 验证 Python 心跳、可用度、flags。
- `FB15_群控调度` 的策略 2 使用智能需求和权重影响外呼分配。
- 智能帧失效时自动退回传统就近调度。

但这只覆盖样题“智能化应用”的一部分，不等于完整覆盖样题所有 PLC 功能。
