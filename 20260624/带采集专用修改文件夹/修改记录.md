## 详细修改

### 1. FB15_群控调度 — 消号冷却楼层化

**问题：** 消号冷却标志是全局Bool，3楼开门消号后，全局800ms内所有楼层的外呼都被阻止登记。且边沿记忆在冷却守卫外部更新，导致上升沿被"吞掉"，外呼永久丢失。

**修复：**
- `上呼冷却中 : Bool` → `Array[1..9] of Bool` + `上呼冷却计数器 : Array[1..9] of Int`
- `下呼冷却中 : Bool` → `Array[2..10] of Bool` + `下呼冷却计数器 : Array[2..10] of Int`
- 冷却管理从两个TON改为每扫描周期计数器递减（80→0，≈800ms）
- 登记条件：`NOT 冷却中` → `NOT 冷却中[#楼层]`
- 消号时只封锁当前楼层：`冷却中[楼层]:=TRUE; 计数器[楼层]:=80`

### 2. FB15_群控调度 — 超时重分配时间基准修复

**问题：** 原代码用扫描计数器（每10扫描累加1），比较对象是秒数配置值。假设扫描周期10ms，配置30秒实际约3秒就超时，时间缩小约10倍。

**修复：**
- 删除 `群控扫描计数器 >= 10` 逻辑
- 新增 `重分配秒脉冲TON`，自复位1秒脉冲(`IN := NOT Q`)
- `IF 重分配秒脉冲TON.Q THEN` → 等待周期每秒累加1，与配置秒数精确对齐

### 3. FB10_本层停靠判断 — 离开确认机制

**问题：** 电梯在某层服务完毕刚起步、仍未脱离平层区时，若本层外呼重新登记，FB10会触发"静止本层捕获"或"运行到站捕获"，导致电梯回到服务状态→再次关门→再次起步→循环往复，最终上下平层信号有一个脱出后无法满足起动条件，电梯卡死。

**修复：**
- 新增 `正在离开本层 : Bool` / `离开时双平层已断 : Bool` 两个Static变量
- 新增离开确认状态机：
  - 梯在运行中 + `当前楼层 = 上次捕获楼层` → 标记"正在离开"
  - "正在离开"且双平层都断开 → 确认"已离开"
  - 楼层计数变化(`当前楼层 <> 上次捕获楼层`) → 清除所有离开标记
- 捕获条件新增：`AND NOT (正在离开本层 AND NOT 离开时双平层已断)`

### 4. FB8_双向初始化 — 检修守卫

**问题：** FB8没有检修模式守卫（FB1/FB3/FB6/FB10/FB12都有）。如果检修在初始化中途被触发：
- BUG A：FC_检修清理清零接触器，但FB8段5的CASE会重新设置接触器，电梯在检修期间试图运动
- BUG B：`当前步骤` 卡在非零值，检修退出后条件C检查 `当前步骤=0` 失败，永远无法重初始化

**修复：**
- 新增段0检修守卫，模式与其他FB一致：
  ```
  IF #电梯.检修信号 THEN
      #当前步骤 := 0;
      清全部接触器;
      RETURN;
  END_IF;
  ```

### 5. FB2_检修控制 — 检修状态门/指示灯管理

**问题：** 
- 检修进入时将开门继电器清零，规则要求"门保持常开"
- 故障指示灯从未被置位

**修复：**
- 段2(进入检修脉冲)：
  - `开门继电器 := FALSE` → `开门继电器 := TRUE`（保持常开）
  - 新增 `故障指示 := TRUE`（点亮故障灯）
- 段3(退出检修脉冲)：
  - 新增 `故障指示 := FALSE`（熄灭故障灯）

### 6. FC_检修清理 — 职责精简

**问题：** FC_检修清理每扫描周期清零 故障指示 和 门继电器，与FB2的管理逻辑冲突，导致：
- 故障灯亮一个周期就被灭
- 门刚被FB2设为常开就被关掉
- FB2段6的人工手动开关门只能维持一个周期

**修复：** 删除以下四行：
- `#电梯.故障指示 := FALSE;`
- `#电梯.开门继电器 := FALSE;`
- `#电梯.关门继电器 := FALSE;`

门继电器和故障指示改由FB2统一管理，FC_检修清理只保留接触器/制动/方向/状态清零。

### 7. FC_输出映射 — 准备就绪检修梯豁免

**问题：** 三部梯全部就绪才输出全局 `准备就绪信号`。如果比赛开始时2号3号处于检修态，1号已初始化完成，但全局就绪永远不亮，EETPRO不派客。

**规则要求：** "初始化时PLC只需等待非检修梯完成初始化，即可输出准备就绪信号。检修梯不应阻塞全局就绪。"

**修复：**
```
原来: 梯[1].就绪 AND 梯[2].就绪 AND 梯[3].就绪
改为: (梯[1].就绪 OR 梯[1].检修) AND (梯[2].就绪 OR 梯[2].检修) AND (梯[3].就绪 OR 梯[3].检修)
```
自锁不变：`IF NOT OUT.准备就绪 THEN ...` — 一旦置TRUE永不复位。

### 8. FB11_称重数据分析 — 阈值下沉UDT

**问题：** 三部梯共享同一套满载阈值/超重阈值（DB_系统配置 中的单值），无法为不同电梯设置不同阈值。

**修复：**
- 删除Input参数 `满载阈值(Real)` 和 `超重阈值(Real)`
- `SCALE_X(MAX := #超重阈值)` → `SCALE_X(MAX := #电梯.超重阈值_kg)`
- `#电梯.满载指示 := 实际载重 > #满载阈值` → `> #电梯.满载阈值_kg`
- `#电梯.超载信号 := 实际载重 >= #超重阈值` → `>= #电梯.超重阈值_kg`

### 9. FB7_单梯控制 — FB11调用精简

**修复：**
```
原来: #载重数据分析(模拟量最大值:=..., 满载阈值:=..., 超重阈值:=..., 电梯:=...)
改为: #载重数据分析(模拟量最大值:=..., 电梯:=...)
```

### 10. OB1 — 每梯阈值注入

**修复：** 在 `段4: 单梯控制循环` 的 FOR 中，CASE 之前新增：
```
"DB_全部电梯".梯数组[#梯号].满载阈值_kg := "DB_系统配置".梯满载阈值[#梯号];
"DB_全部电梯".梯数组[#梯号].超重阈值_kg := "DB_系统配置".梯超重阈值[#梯号];
```
每扫描周期将配置数组同步到各梯UDT。

## TIA Portal 手动操作

代码修改完成后，需在博图中完成以下结构变更（.txt无法体现）：

| 位置 | 操作 |
|------|------|
| `UDT_单梯状态` | 新增字段 `满载阈值_kg : Real`、`超重阈值_kg : Real` |
| `DB_系统配置` | 新增 `梯满载阈值 : Array[1..3] of Real`（默认 `[450.0, 450.0, 450.0]`） |
| `DB_系统配置` | 新增 `梯超重阈值 : Array[1..3] of Real`（默认 `[600.0, 600.0, 600.0]`） |

旧的 `满载阈值_kg`（单值Real）和 `超重阈值_kg`（单值Real）可保留或删除。

## 修改后的检修完整时序

```
检修进入（任意时刻）:
  FB2段2: 停车 + 开门继电器=TRUE(门常开) + 故障指示=TRUE + 准备就绪=FALSE
  FB8段0: 当前步骤:=0 + 清接触器 + RETURN
  FC_检修清理: 清接触器/制动/方向/状态（不再动门继电器和故障指示）

检修中:
  接触器: FC_检修清理每周期清零（冗余安全）
  门:    保持常开（FB2设的），FB2段6可人工手动开关门
  指示灯: 故障指示灯常亮
  群控:   FB15过滤故障梯，不参与调度分配

检修退出:
  FB2段3: 清接触器/门继电器 + 故障指示:=FALSE
  FB7:    检修=FALSE → 跳过FC_检修清理
  FB8段0: 检修=FALSE → 放行
  FB8条件C: NOT检修 AND 上次记忆 AND 步=0 → 触发!

退出后重初始化:
  FB8步10/20: 门锁未闭合 → 关上门 → 等门锁闭合 → 低速起动寻端站
  FB8步11-13或21-23: 全程低速、限位保护、精准平层
  完成: 准备就绪:=TRUE → 重新加入群控

全局:
  准备就绪信号: 自锁保持TRUE（单梯重初始不断开）
  整个过程中EETPRO持续派客评分不受影响
```
