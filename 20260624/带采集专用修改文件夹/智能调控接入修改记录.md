# 智能调控接入修改记录

日期：2026-06-24

## 目标

在新工程中保留原 PLC 安全、门控、检修、超载和单梯控制逻辑，只新增 Modbus 智能接口：

- `DB_Modbus采集.寄存器[0..39]`：PLC 写，Python 读，用于采集状态。
- `DB_Modbus采集.寄存器[40..79]`：Python 写，PLC 读，用于智能调度建议。

Python 不直接控制接触器、门、制动和安全输出，只影响群控外呼分配。

## 已修改文件

### DB 模块

- `DB_Modbus采集.db`
  - `寄存器` 从 `Array[0..39] of Word` 扩展为 `Array[0..79] of Word`。

- `DB_群控状态.db`
  - 新增智能帧状态、Python 心跳、预测需求和智能权重变量。

- `DB_系统配置.db`
  - `群控调度策略` 注释更新：
    - `1` = 传统就近调度
    - `2` = Python 智能调度，失效自动退回传统就近调度
  - 新增：
    - `智能心跳超时 : Time`
    - `智能可用度最小值 : Int`

- `智能帧验证_DB.db`
  - 新增 `FB_智能帧验证` 的实例 DB。

### TXT 文件

- `FB17modbus_tcp_new.txt`
  - `MB_HOLD_REG` 从 `BYTE 80` 改为 `BYTE 160`。

- `FB_智能帧验证.txt`
  - 新增智能帧验证逻辑。
  - 验证魔数、协议版本、首尾帧号、flags、可用度和心跳。
  - 将 Python 写回的需求评分和权重复制到 `DB_群控状态`。

- `FB15_群控调度.txt`
  - 新增策略 2。
  - 策略 2 只改变外呼分配评分，不直接控制硬件动作。
  - 智能帧无效时自动退回原策略 1。
  - 增加 `群控扫描计数器` 自增，方便 Python 判断 PLC 数据正在刷新。

- `OB1.txt`
  - 调用顺序调整为：

```scl
"FC_输入映射"();
"tcp"();
"智能帧验证_DB"();
"群控调度_DB"();
// 单梯控制循环
// 电机启动信号汇总
"FC_输出映射"();
"caiji"();
```

## Python 写回寄存器

| 地址 | 含义 |
|---:|---|
| 40 | 魔数 `16#A55A` |
| 41 | 协议版本，当前为 `1` |
| 42 | 帧序号 |
| 43 | 心跳 |
| 44 | 预测序号 |
| 45 | flags，位 0..5 必须为 1，位 6 不能为 1 |
| 46 | 可用度，0..1000 |
| 47 | 主场景 |
| 48..56 | 上行 1..9 层需求评分 |
| 57..65 | 下行 2..10 层需求评分 |
| 66..72 | 7 类场景概率，PLC 第一版暂不直接使用 |
| 73 | 等待权重 |
| 74 | 距离权重 |
| 75 | 反向惩罚权重 |
| 76 | 启停惩罚权重 |
| 77 | 预测权重 |
| 78 | checksum，PLC 第一版暂不强制 |
| 79 | 帧序号重复 |

## 博图操作顺序

1. 更新 `DB_Modbus采集.db`，重新生成/下载 DB。
2. 更新 `DB_群控状态.db`。
3. 更新 `DB_系统配置.db`。
4. 新建 FB：`FB_智能帧验证`，把 `FB_智能帧验证.txt` 内容复制进去。
5. 新建实例 DB：`智能帧验证_DB`，类型为 `FB_智能帧验证`。
6. 更新 `FB17modbus_tcp_new.txt`。
7. 更新 `FB15_群控调度.txt`。
8. 更新 `OB1.txt`。
9. 编译通过后先保持 `DB_系统配置.群控调度策略 := 1` 测试传统调度。
10. Python 控制程序稳定写入后，再把 `群控调度策略` 改为 `2`。
